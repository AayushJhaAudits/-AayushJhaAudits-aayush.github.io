üîç Vulnerability Explanation

Solmate favors minimalism and gas-efficiency, but missing these basic checks can cause catastrophic failures in production. Funds could be sent to address(0) and burned, or transferred silently even when users had insufficient balance.

‚úÖ Secure, Gas-Efficient Fix

I implemented two custom errors and added conditional checks with revert:

error InsufficientBalance();
error TransferToZeroAddress();

function transfer(address to, uint256 amount) public returns (bool) {
    if (balanceOf[msg.sender] < amount) revert InsufficientBalance();
    if (to == address(0)) revert TransferToZeroAddress();

    unchecked {
        balanceOf[msg.sender] -= amount;
        balanceOf[to] += amount;
    }

    emit Transfer(msg.sender, to, amount);
    return true;
}

This follows Solmate‚Äôs gas-optimized design while securing basic user safety.
